## 🔄 交互逻辑设计

### 页面加载和数据获取
```javascript
// 页面初始化流程
1. 加载客户列表 (分页，默认显示50条)
2. 加载筛选条件统计数据
3. 设置默认筛选 (最近3个月的客户)
4. 如果URL包含客户ID，自动选中并显示详情
```

### 筛选和搜索逻辑
```javascript
// 实时筛选机制
搜索框: 防抖300ms，支持姓名/电话/标签搜索
状态筛选: 多选，实时更新列表
标签筛选: 多选，支持标签组合查询
时间筛选: 单选，预设选项 + 自定义范围
预算筛选: 多选，支持自定义范围输入
```

### 列表交互
```javascript
// 客户列表操作
单击行: 在右侧面板显示客户详情
双击行: 跳转到聊天页面
checkbox: 多选进行批量操作
操作按钮: 
  - 💬 直接跳转聊天
  - 📞 记录通话/创建通话任务
  - 📋 快速编辑客户信息
```

### 详情面板操作
```javascript
// 右侧面板功能
编辑信息: 内联编辑，实时保存
添加标签: 下拉选择 + 快速创建新标签
记录备注: 支持富文本，自动保存草稿
快速操作: 一键执行常用操作
时间线: 支持筛选显示特定类型活动
```

## 📱 响应式设计

### 桌面端 (≥1200px)
```
三栏完整布局: 筛选面板(300px) + 列表(自适应) + 详情(350px)
```

### 平板端 (768px - 1199px)
```
两栏布局: 列表(70%) + 详情面板(30%)
筛选面板: 收缩为顶部筛选条
```

### 手机端 (<768px)
```
单栏布局: 
- 客户列表占满屏幕
- 筛选功能: 抽屉式侧滑面板
- 客户详情: 模态窗口覆盖显示
```
